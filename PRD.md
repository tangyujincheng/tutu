# 兔兔信件小屋 - 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 项目名称
兔兔信件小屋 (Bunny Letter Game)

### 1.2 项目背景
主人喜欢可爱的小兔子，希望有一个可以和AI兔兔写信互动的温馨小应用。每次主人给兔兔写信，兔兔都会用温暖可爱的语气回信；如果主人很久不写信，兔兔也会主动想念主人并主动写信。

### 1.3 目标用户
单用户场景 - 仅为产品主人个人使用，不需要多用户支持。

---

## 2. 核心需求

### 2.1 需求一：AI自动回信功能 ✅
**需求描述：**
- 主人可以在前端给小兔子写信
- 信件发送后，后端调用AI自动生成一封可爱的回信
- 回信保存在系统中，主人可以随时查看

**验收标准：**
- [x] 用户输入信件内容后点击发送，成功保存用户信件
- [x] 后端自动调用AI生成回信并保存
- [x] 前端可以看到AI生成的回信内容
- [x] AI回信语气符合小兔子可爱软萌的设定

### 2.2 需求二：未读消息提示 ✅
**需求描述：**
- 当有兔兔的未读回信时，需要显示小红点提示
- 用户打开信箱查看后，小红点消失

**验收标准：**
- [x] 未读回信数量 > 0 时显示小红点
- [x] 用户打开信箱后自动标记所有未读为已读
- [x] 标记已读之后小红点消失

### 2.3 需求三：超时主动发信 ✅
**需求描述：**
- 如果超过2小时主人没有给兔兔写信，且最后一封信不是兔兔发的
- 兔兔需要主动给主人写信（使用AI生成内容）

**验收标准：**
- [x] 系统定时检查距离上次用户发信的时间间隔
- [x] 满足条件（>2小时，最后一封不是兔兔发信）时自动触发AI发信
- [x] 主动发信后会显示未读提示，用户可以看到来信

### 2.4 需求四：安全存储AI配置 ✅
**需求描述：**
- AI接口的API密钥等敏感信息必须安全存储
- 绝对不能暴露到前端代码中
- 提供后台配置页面，方便修改AI配置和提示词

**验收标准：**
- [x] API密钥加密存储在后端数据库
- [x] 前端API无法获取任何敏感信息
- [x] 提供独立的后台配置页面，可以修改：
  - API地址
  - API密钥
  - 模型ID
  - 系统提示词（AI回信风格）
- [x] 密钥在后台也仅显示掩码，完整密钥不暴露

### 2.5 需求校准：拼字母游戏 → AI兔兔信件 ✅
**校准说明：**
- 最初的纯前端拼字母预览项目已经完成开发并修复了预览不更新的Bug
- 产品方向调整为更有情感价值的AI兔兔写信互动应用
- 保留整体可爱温馨的视觉风格，核心交互从拼字母改为写信互动

## 3. 交互流程

```
用户打开页面
  ├─► 查看是否有未读回信 → 有则显示小红点
  ├─► 用户点击"查看信箱" → 打开信箱弹窗
  │   └─► 显示所有信件列表 → 点击信件查看详情 → 自动标记未读
  └─► 用户点击"给兔兔写信" → 打开写信弹窗
      └─► 输入内容 → 发送 → 后端保存并AI回信 → 返回结果 → 更新信箱

定时后台检查：
  每分钟检查一次
  ├─► 如果 距离上次用户发信 > 2小时
  ├─► 并且 最后一封信不是兔兔发的
  └─► 触发兔兔主动发信 → AI生成内容保存 → 更新未读计数
```

## 4. 视觉设计要求

### 4.1 整体风格
- 温馨治愈，软萌可爱
- 暖色调为主，符合兔兔小屋的感觉
- 交互流畅，动画自然

### 4.2 核心元素
- 主体：一只可爱的兔兔插图
- 交互按钮：查看信箱、写信
- 弹窗：信箱列表、写信弹窗
- 未读提示：小红点徽章

## 5. 技术约束

### 5.1 部署环境
- 部署服务器：`14.103.60.103`
- 部署路径：`/var/www/bunny-letter-game/`
- 使用PM2守护Node.js进程
- Nginx反向代理到Node.js服务

### 5.2 技术栈要求
- **后端**：Node.js + Express
- **数据库**：SQLite（轻量单文件，适合单用户）
- **ORM**：Prisma
- **AI**：火山方舟ARK大模型API
- **前端**：原生JavaScript + CSS，不需要框架

### 5.3 安全要求
- API密钥必须加密存储
- 敏感信息绝对不能暴露到前端
- 加密密钥从环境变量读取
